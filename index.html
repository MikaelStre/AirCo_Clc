How to use this app

What the Engineering Suite does, how the pages work together, and what it can’t do yet.

What it is

The Baltic Yachts Engineering Suite is a single web-app that gathers the early-stage system calculations for a yacht in one place. Every calculation is tied to an active project, and the consumers each system creates (pumps, fans, compressors…) are collected automatically into a single Electrical Load Balance. It runs in any modern browser, works offline once loaded, and can be installed as an app on a phone, tablet or desktop.
Created by Mikael Strengell, Head of Engineering, Baltic Yachts.

Getting started in 4 steps

  1. Create a project. On the Home page open Projects → + Create Project and enter the yacht name, project number, type, class society and (optionally) flag. The active project is shown in the chip at the top-right of every page.
  2. Open a discipline. Pick a tile — Systems (AC, Seawater, Bilge, Engine-room Ventilation, Water & Waste, Engine/Thruster) or Electrical.
  3. Fill in the inputs. Each calculator sizes its equipment and lists the electrical consumers it produces.
  4. Check the Electrical Load Balance. It automatically totals the consumers from every calculation so you see the whole-boat picture in one place.

Saving & your data

  • Projects are saved in this browser, on this device (local storage). Use Save current on Home to store your work, and switch between saved projects from the same list.
  • Because data lives in the browser, it is not shared between devices or colleagues, and clearing the browser’s site data will remove saved projects. Keep important cases backed up.
  • Use the print button (top-right) to produce a PDF / paper copy of any page.
  • Install it as an app: your browser may offer “Install” / “Add to Home Screen”. Once installed it opens full-screen and works offline.

Current limitations

  • Preliminary estimates only. Results, equipment selections and rule compliance must be independently checked by a qualified engineer against the applicable class society, flag administration and manufacturer documentation before use.
  • Not every discipline is built yet. Interior (joinery, weights, furnishing) and Mechanical (steering, hydraulics, deck machinery) are placeholders. The Propeller calculation currently opens an external Excel workbook rather than running in-app.
  • The electrical supplier library is still being expanded. AC consumers feed the load balance automatically; charger / inverter / combi units, 24 V–48 V and high-voltage (800 V) battery options are being added.
  • Catalogue & rule data are starting points. AC unit catalogues, FAMU data and the class/flag rule sets should be verified against the latest manufacturer datasheets and the governing rules for your project.
  • Single-user, local data. No cloud sync, version history or multi-user editing — each browser holds its own copy.

Class & Flag

How classification societies and flag administrations shape a Baltic project — and how the Suite applies their rules.

The two rule sets

Every yacht is engineered to two overlapping sets of requirements:
  • Class society — an independent body that sets and verifies the technical standards for the design, construction and survey of the hull, machinery and systems. Class covers how the boat is engineered.
  • Flag state (flag administration) — the country the yacht is registered under. It sets the statutory requirements (safety, pollution prevention, manning, certification) layered on top of class. Flag covers how the boat is operated and certified.

Class societies in the Suite

Pick the society when you create a project. The calculation pages tag their outputs with that society’s rule references (for example, bilge pipe sizing to RINA / DNV‑GL). The Suite’s calculations are currently reconciled against DNV GL and RINA:
DNV GL RINA Other (record only — no society rules applied)
Not yet available: Bureau Veritas (BV), ABS and Lloyd’s Register (LR). These rule sets have not been reconciled against the Suite’s calculations yet, so they are not selectable when creating a project. They will be added once verified.

Flag states in the Suite

The flag adds its own rule book on top of class. The Suite groups flags the way their rules align:
Red Ensign Group — British registries sharing the UK / Large Yacht Code (LY3) framework:
United Kingdom · Cayman Islands · Gibraltar · Bermuda · Isle of Man · British Virgin Islands · Jersey · Guernsey · Falkland Islands · St Helena · Anguilla · Montserrat · Turks & Caicos Islands
Malta — common EU flag with its own commercial yacht code.
Other / international — any flag outside the groups above; the standard rule set applies.

How the Suite applies them

  • Setting the class and flag on a project assembles the active rule set: class rules + flag rules + always‑applicable standards.
  • The active rule set is shown on the Home page for the open project, grouped by system (air conditioning, ventilation, bilge, water, electrical…).
  • Use Manage Rules on Home to add, edit, import or export rules, so the framework tracks the exact references your projects rely on.

Important

The rule references in the Suite are a reference aid to keep calculations traceable — they do not replace the governing class and flag rule books, which must be checked by a qualified engineer for each project.
